# AI 编程方法步骤——跨 IDE 提示词速查卡

> 适用于 Codebuddy、Tongyi Lingma 等不支持斜杠命令的 IDE。
> Claude Code / Trae 用户直接使用 `/spec` `/plan` `/review` `/test`，无需此卡。

---

## 步骤一：需求规格（等效 `/spec`）

```
请按以下五要素为我生成结构化需求规格，每项不超过 150 字：

1. 背景描述：要解决什么问题、面向什么用户
2. 约束条件：不做什么、边界在哪里
3. 输入格式：数据从哪来、字段定义、数据量级
4. 期望输出：产出什么、格式是什么
5. 验收标准：怎么判断"对"——可量化的指标

需求：[在此填写你的需求描述]
```

---

## 步骤二：任务拆解（等效 `/plan`）

```
请将以下需求规格拆解为开发任务，输出：
- 模块列表（每个模块一句话职责说明）
- 依赖排序（哪个先做、哪个后做、理由）
- 各模块接口定义（输入参数 + 返回值类型）
- 每个模块的验收标准

[粘贴步骤一产出的需求规格]
```

---

## 步骤三：质量审查（等效 `/review`）

```
请从四个维度审查以下代码，每个问题标注严重等级：
CRITICAL（必须修复）/ HIGH（应当修复）/ MEDIUM（建议修复）/ LOW（可选优化）

审查维度：
1. 可维护性：圈复杂度、重复代码、命名清晰度
2. 可靠性：错误处理、边界条件、空值处理
3. 安全性：输入验证、依赖库风险
4. 性能：时间复杂度、内存占用、I/O 效率

特别关注 IoT 场景特有问题：数据重传导致的重复记录、欠费断电误判为计量故障

[粘贴待审查代码]
```

---

## 步骤四：测试生成（等效 `/test`）

```
请为以下功能生成 pytest 测试用例，按四层策略覆盖：

1. 正常路径：典型输入 → 期望输出
2. 边界值：[在此填写你的边界条件，AI 不会自动知道]
3. 异常路径：非法输入、空数据、格式错误
4. 组合场景：多条件叠加（如同时存在重复时戳 + 零用量）

输出格式：pytest 风格，含 AAA 结构注释（Arrange / Act / Assert）

功能描述：[粘贴函数签名或功能说明]
```

---

> **提示**：步骤三和步骤四中括号内的内容需要你自己填写——这正是"AI 想不到、只有你懂"的部分。
