## 操作步骤

### Step 1：用 Claude Code 生成需求规格

启动 Claude Code（`claude`），输入以下 prompt：

```
我是联通物联网运营团队的工程师。我们每半小时从智能电表收集一次用电数据
（字段：meter_id, timestamp, kwh, cumulative_kwh），需要自动检测三类异常：

1. 欠费断电：连续 7 天以上零用量，且累计用电量冻结不变
2. 计量故障：连续 7 天以上零用量，恢复后累计用电量出现不连续跳变
3. 窃电行为：夜间（00:00-06:00）用电持续高于白天（09:00-17:00）1.5 倍以上

输出要求：CSV 格式告警清单，包含设备编号、异常类型（中英文）、置信度、判断依据、建议动作。
验收标准：漏报率 < 1%，误报率 < 5%，运行时间 < 30 秒（100 台设备 × 10 天数据）。

请帮我生成一份结构化需求规格文档，包含五个部分：角色、场景、输入、输出、验收标准。
```

**预期输出**：Claude Code 输出包含五要素的 Markdown 规格文档。

**验收检查**（逐项打勾）：
- [ ] **角色**：描述使用者（联通物联网运营工程师）
- [ ] **场景**：描述触发条件（半小时粒度数据 + 三类异常检测）
- [ ] **输入**：4 个字段，含 cumulative_kwh 及其说明
- [ ] **输出**：告警清单 6 个字段（meter_id / anomaly_type / anomaly_type_cn / confidence / evidence / recommended_action）
- [ ] **验收标准**：漏报率 / 误报率 / 运行时间三项指标

---

### Step 2：将规格写入文件

在 Claude Code 中继续输入：

```
请将上面的规格写入 spec.md 文件。
```

**验收检查**：项目根目录下出现 `spec.md` 文件。

### Step 3：逆练法——从结果反推 prompt

逆练法的核心思路：如果你的 prompt 足够精确，别人看到你的输出后，应该能还原出你的 prompt。

打开 `data/anomaly_labels.csv`，可以看到已知异常标签（这是"答案 key"，课程结束后才对照）：

```
meter_id,anomaly_type,anomaly_type_cn,meter_status
EM-003,overdue_shutoff,欠费断电,disconnected
EM-005,metering_fault,计量故障,replaced
EM-007,electricity_theft,窃电行为,suspected
EM-008,single_day_spike,窃电嫌疑,suspected
EM-010,data_quality,数据质量,normal
```

在 Claude Code 中输入：

```
我有一份电表异常检测结果：
- EM-003：欠费断电
- EM-005：计量故障
- EM-007：窃电行为
- EM-008：窃电嫌疑（单日跳变）

请根据这份结果，反推生成它所需的检测规则和关键数据特征。
```

**预期输出**：Claude Code 推断出三条规则（连续零用量阈值 / 峰谷比阈值 / 单日跳变倍数）。

**验收检查**：推断出的规则是否与 Step 1 规格中的验收标准一致？

> **讨论题**：EM-003（欠费断电）与 EM-005（计量故障）都是连续零用量，靠什么区分？  
> 答案：看恢复后的 `cumulative_kwh` 行为——欠费补缴后继续累积，换表后从较低值重置。

## 为 Skill 贡献了什么

| Skill 部分 | 来自本单元 |
|-----------|-----------|
| ① 触发条件 | Step 1 规格中的"角色 + 场景" |
| ④ 输出规范 | Step 1 规格中的"输出格式 + 验收标准" |

---

## 常见卡点与解决方案

| 卡点 | 解决方案 |
|------|---------|
| 输出规格格式混乱 | Prompt 末尾加：**"请用 Markdown 表格格式输出每个部分"** |
| 逆练推断的规则与原规格不一致 | 说明 prompt 不够精确，返回 Step 1 补充具体阈值数值 |
| 不清楚 cumulative_kwh 如何区分两类零用量 | 欠费断电：零值期后累计量继续上升；计量故障：恢复后累计量跳变到较低值 |

## 本单元小结

- 需求规格五要素是 prompt 工程的核心框架，也是 Skill 的设计蓝图
